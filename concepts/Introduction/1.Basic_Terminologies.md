# 📘 Data Structures & Basic Terminologies (DSAP)

This document covers the fundamental concepts of Data, Information, Data Organization, Data Structures, and Abstract Data Types (ADT). It is structured for **exam preparation** and **quick revision**.

---

## 📌 1. DATA

### ✅ Definition
Data is a collection of **raw facts and figures** that are unprocessed and do not have any meaningful interpretation on their own.

### ✅ Explanation
- Data acts as the **input** for any system.
- It can exist in multiple forms:
  - **Numbers** → `100`, `78.5`
  - **Characters** → `A`, `B`
  - **Strings** → `Tanmay`
- By itself, data does **not provide meaning**.

### ✅ Example

78, 82, 90 → Just numbers (no meaning)


### ✅ Key Points
- Raw and unprocessed
- No context or meaning
- Used as input

---

## 📌 2. INFORMATION

### ✅ Definition
Information is **processed, organized, or structured data** that is meaningful and useful.

### ✅ Explanation
- Information is obtained after **processing data**.
- It helps in:
  - Decision making
  - Analysis
  - Understanding

### ✅ Example

Data: 78, 82, 90
Information: Average = 83.3


### ✅ Key Points
- Meaningful
- Processed data
- Output of a system

---

## 📌 3. DATA ORGANIZATION

### ✅ Definition
Data organization refers to the **systematic arrangement of data** in a structured manner so that it can be accessed and processed efficiently.

---

### 🔥 Levels of Data Organization (VERY IMPORTANT)

#### 🔹 1. Character (Smallest Unit)
- Smallest unit of data
- Examples: `'A'`, `'5'`, `'#'`
- Represented using **ASCII/Unicode**

---

#### 🔹 2. Field
- Group of characters representing a **single attribute**
- Examples:
  - `Name = "Tanmay"`
  - `Age = 23`

---

#### 🔹 3. Record
- Collection of related fields
- Example:

Name = Tanmay
Age = 23
Marks = 85

- Represents **one entity**

---

#### 🔹 4. File
- Collection of similar records
- Example:

Student File:
Record1
Record2
Record3


---

#### 🔹 5. Database
- Collection of related files
- Example:
- Student File
- Teacher File
- Course File

---

### 🎯 Diagram (IMPORTANT FOR EXAM)


Character → Field → Record → File → Database


---

## 📌 4. DATA STRUCTURE

### ✅ Definition
A Data Structure is a method of organizing and storing data in a way that enables **efficient access and modification**.

### ✅ Explanation
- Helps in:
  - Efficient storage
  - Fast processing
  - Easy access

---

### 🔹 Types of Data Structures

#### 1. Primitive Data Structures
- Basic data types provided by programming languages

**Examples:**

int, float, char, boolean


**Key Points:**
- Store single values
- Simple and built-in

---

#### 2. Non-Primitive Data Structures

##### (a) Linear Data Structures
- Elements arranged sequentially

**Examples:**

Array, Linked List, Stack, Queue


**Key Points:**
- Easy traversal
- Each element has a unique predecessor and successor

---

##### (b) Non-Linear Data Structures
- Elements arranged hierarchically

**Examples:**

Tree, Graph


**Key Points:**
- Complex relationships
- Used in advanced applications

---

## 📌 5. ABSTRACT DATA TYPE (ADT)

### ✅ Definition
An Abstract Data Type (ADT) is a **logical model** of a data structure that defines operations without specifying implementation details.

### ✅ Explanation
- Focuses on **what to do**, not **how to do**
- Implementation is hidden (**encapsulation**)

---

### 🔹 Example: Stack ADT

**Operations:**

push() → Insert element
pop() → Remove element
peek() → View top element


---

### 🔹 Implementation Methods

#### 1. Using Array
- Uses contiguous memory
- Fixed size

#### 2. Using Linked List
- Uses dynamic memory
- Flexible size

---

### 🎯 Key Points (VERY IMPORTANT)
- Hides implementation details
- Improves modularity
- Same ADT → Multiple implementations
